# 五维度框架入门指南

> 开始使用五维度人生管理系统

---

## 第一步：理解框架（15分钟）

在开始之前，先理解五维度框架的基本概念：

```
方向：我要去哪里？（愿景、目标）
行动：如何到达那里？（任务、习惯）
认知：如何理解正在发生的事？（课题、洞察）
反思：学到了什么？（复盘、调整）
资源：凭什么支撑？（健康、人脉、工具）
```

**推荐阅读顺序**：
1. `OVERVIEW.md` - 框架总览（本文件）
2. `direction/VISION.md` - 从愿景开始
3. `action/TASKS.md` - 然后建立行动系统

---

## 第二步：建立愿景（30分钟）

愿景是整个系统的起点。没有清晰的愿景，行动就会失去方向。

### 练习：构建你的愿景

```
回答以下问题，不需要完美，诚实即可：

1. 回顾你人生的高峰时刻，什么让你感到最满足？
   _____________________________________________

2. 如果你有足够的时间和金钱，你最想做什么？
   _____________________________________________

3. 你希望10年后自己在做什么？
   _____________________________________________

4. 如果你只能做一件事让世界变得不同，那是什么？
   _____________________________________________

5. 描述你理想中普通的一天，从早到晚。
   _____________________________________________
```

### 输出：愿景陈述

基于以上练习，写下你的愿景陈述：

```
我的愿景：

在____岁时，我想____

我想成为____的人

这对我意味着____
```

## 第三步：建立行动系统（1小时）

### 步骤3.1：收集

将所有盘旋在你脑中的事项转移到外部系统：
- 打开一个笔记本或数字工具
- 写下所有你想做、应该做、必须做的事情
- 持续写，直到大脑清空

### 步骤3.2：处理

对每个事项做出决定：
- **2分钟内能完成？** → 立即做
- **需要多步？** → 归入"项目"
- **需要等待？** → 归入"等待"
- **不需要做？** → 删除

### 步骤3.3：组织

建立你的系统结构：
```
收件箱 → 项目清单 → 下一步行动 → 日历

项目示例：
- 完成Q2报告（需要多步）
- 学习摄影（长期课题）

下一步行动示例：
- 给客户发邮件
- 完成报告第一章
- 搜索摄影课程
```

## 第四步：建立习惯追踪（30分钟）

### 选择你的核心习惯

```
问自己：如果只能养成3个习惯，应该是什么？

推荐从以下选择：
□ 早起（或者设定一个固定的起床时间）
□ 每日运动
□ 每日阅读
□ 每日冥想
□ 每日复盘
□ 早睡

选择你的3个：____、____、____
```

### 建立习惯触发

```
对于每个习惯，设计触发：

习惯1：____
触发：做完____之后，我就做____

习惯2：____
触发：做完____之后，我就做____

习惯3：____
触发：做完____之后，我就做____
```

## 第六步：评估资源（30分钟）

### 健康基础

```
回答以下问题：

1. 最近一次体检是什么时候？结果如何？
   _____________________________________________

2. 你目前的运动频率如何？
   □ 几乎不 □ 每周1-2次 □ 每周3次以上

3. 你的睡眠质量如何？（1-10分）
   □ 1-3 □ 4-6 □ 7-8 □ 9-10

4. 你目前最大的健康挑战是什么？
   _____________________________________________
```

### 人脉基础

```
回答以下问题：

1. 你最核心的3段关系是谁？
   1. ____  2. ____  3. ____

2. 你上次联系他们是什么时候？
   1. ____天前  2. ____天前  3. ____天前

3. 你有多少"点头之交"但很久没联系？
   □ 0-10  □ 10-30  □ 30-50  □ 50+
```

### 订阅基础

```
回答以下问题：

1. 你每月在订阅服务上花费多少？
   □ <100元 □ 100-300元 □ 300-500元 □ 500+元

2. 你知道自己订阅了哪些服务吗？
   □ 完全清楚  □ 大概知道  □ 不太清楚

3. 是否有订阅你几乎不用但还在付费的？
   □ 是（哪些：____）  □ 否
```

## 第七步：整合运行（持续）

### 第一周：适应

```
每日任务：
□ 早上看日历，确认今日安排
□ 傍晚做每日复盘
□ 记录任何系统问题

每日评估：
□ 系统是否过于复杂？
□ 哪个部分感觉最好？
□ 哪个部分感觉困难？
```

### 第二周：调整

```
基于第一周反馈调整：
□ 简化过于复杂的部分
□ 加强感觉困难的部分
□ 补充缺失的部分

周复盘：
□ 本周进展如何？
□ 下周要改进什么？
```

### 第三周及以后：固化

```
形成习惯：
□ 每天同一时间做复盘
□ 每周同一时间做周检视
□ 保持系统干净

每月审视：
□ 系统是否仍然适合？
□ 是否有需要调整的地方？
□ 是否有新的需求？
```

## 常见问题

### Q1：需要多少时间？

```
起步时间：
- 理解框架：15分钟
- 建立愿景：30分钟
- 建立行动系统：1小时
- 建立习惯：30分钟
- 建立复盘：20分钟
- 评估资源：30分钟
总计：约2.5小时

日常维护：
- 每日复盘：10-15分钟
- 周检视：30-60分钟
- 月审视：1-2小时
```

### Q2：需要什么工具？

```
起步建议（最简单的组合）：
- 纸质笔记本（复盘+快速记录）
- 手机备忘录或简单清单App（日历）
- 智能手表（健康追踪，可选）

进阶工具：
- Notion或Obsidian（知识管理）
- Todoist或滴答清单（任务管理）
- Google Calendar或日历App（日程）

不需要一开始就完美，从简单开始。
```

### Q3：如何保持坚持？

```
关键策略：

1. 从小开始
   - 不要一开始就建立复杂的系统
   - 先从1-2个习惯/系统开始

2. 设置提醒
   - 在日历上设置复盘提醒
   - 固化时间而非靠意志力

3. 接受中断
   - 中断不等于失败
   - 重新开始即可

4. 定期回顾
   - 每周检视进展
   - 每月审视系统

5. 找到支持
   - 有人一起坚持更容易
   - 找到你的"问责伙伴"
```

### Q4：感觉太复杂怎么办？

```
简化原则：

1. 删除非必需的
   - 如果不用，就删掉
   - 系统服务你，不是你服务系统

2. 合并相似功能
   - 一个工具能做的不要用两个

3. 减少频率
   - 每天太累就改为每周3次
   - 周复盘太久就缩短到30分钟

4. 接受"足够好"
   - 不需要完美的系统
   - 需要能用的系统
```
