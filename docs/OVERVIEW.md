# 五维度框架概述

> MeOS 人生管理系统的核心理论框架

---

## 什么是五维度框架

五维度框架是一种系统化的人生管理方法论，将人生管理的要素划分为五个相互关联的维度：

- **方向 (Direction)**：定义你要去哪里
- **行动 (Action)**：确保你真正到达那里
- **认知 (Cognition)**：理解你在做什么
- **反思 (Reflection)**：从经历中提取智慧
- **资源 (Resources)**：为系统提供能量支撑

这五个维度不是独立的，而是形成一个动态循环系统，每个维度相互影响、相互支撑。

## 五维度详解

### 第一维度：方向 (Direction)

回答："我要去哪里？"

方向维度包含四个子维度：

| 子维度 | 核心问题 | 产出 |
|--------|----------|------|
| 愿景 (Vision) | 我想要什么样的人生？ | 人生愿景、长期图景 |
| 领域 (Domains) | 生命中需要管理哪些领域？ | 领域优先级、关注度分配 |
| 目标 (Goals) | 我要达成什么？ | 可衡量的阶段成果 |
| 心态 (Mindset) | 我如何思考？ | 内在信念系统 |

**关键原则**：方向优先于行动。没有清晰方向的行动是盲目的。

**核心文件**：
- `direction/VISION.md` - 愿景构建方法论
- `direction/DOMAINS.md` - 八领域管理模型
- `direction/GOALS.md` - OKR目标管理法
- `direction/MINDSET.md` - 五大核心心态

---

### 第二维度：行动 (Action)

回答："如何做到？"

行动维度包含三个子维度：

| 子维度 | 核心问题 | 产出 |
|--------|----------|------|
| 待办 (Tasks) | 我需要做什么？ | 任务清单、下一步行动 |
| 习惯 (Habits) | 我需要自动化什么？ | 自动执行的行为系统 |
| 日程 (Calendar) | 我什么时候做？ | 时间块、深度工作时间 |

**关键原则**：执行优先于完美。小步持续胜过间歇性冲刺。

**核心文件**：
- `action/TASKS.md` - GTD任务管理法
- `action/HABITS.md` - 习惯养成系统
- `action/CALENDAR.md` - 时间块管理法

---

### 第三维度：认知 (Cognition)

回答："如何理解？"

认知维度包含三个子维度：

| 子维度 | 核心问题 | 产出 |
|--------|----------|------|
| 课题 (Projects) | 我在研究什么？ | 知识体系、深度理解 |
| 洞察 (Insights) | 我学到了什么？ | 个人智慧、可执行洞见 |
| 阅读 (Reading) | 我在输入什么？ | 知识储备、信息网络 |

**关键原则**：知识需要转化才能产生价值。阅读是输入，洞察是产出。

**核心文件**：
- `cognition/PROJECTS.md` - 课题研究方法
- `cognition/INSIGHTS.md` - 洞察提取系统
- `cognition/READING.md` - 高效阅读方法论

---

### 第四维度：反思 (Reflection)

回答："我学到了什么？"

反思维度包含两个子维度：

| 子维度 | 核心问题 | 产出 |
|--------|----------|------|
| 每日反思 (Daily Review) | 今天发生了什么？ | 日志、模式识别 |
| 周期复盘 (Periodic Review) | 这个周期表现如何？ | 周/月/季度洞察 |

**关键原则**：未经审视的经验只是经历。反思将数据转化为智慧。

**核心文件**：
- `reflection/DAILY_REVIEW.md` - 每日反思模板
- `reflection/PERIODIC_REVIEW.md` - 周期复盘方法论

---

### 第五维度：资源 (Resources)

回答："凭什么支撑？"

资源维度包含三个子维度：

| 子维度 | 核心问题 | 产出 |
|--------|----------|------|
| 订阅 (Subscriptions) | 我用什么工具？ | 工具优化、成本控制 |
| 人脉 (Network) | 谁在支持我？ | 关系资产、社会资本 |
| 健康记录 (Health Records) | 我的状态如何？ | 能量追踪、健康仪表盘 |

**关键原则**：失去健康，其他维度无从谈起。资源是所有维度的燃料。

**核心文件**：
- `resources/SUBSCRIPTIONS.md` - 订阅管理方法
- `resources/NETWORK.md` - 人脉建设系统
- `resources/HEALTH_RECORDS.md` - 健康追踪体系

## 维度间关系

### 循环关系

```
方向 → 行动 → 认知 → 反思 → 方向
  ↑                        │
  └────────────────────────┘
            ↓
         资源
```

- **方向** 设定目标，指引 **行动** 的优先级
- **行动** 产生经验，滋养 **认知** 的积累
- **认知** 深化理解，反馈优化 **方向** 的准确性
- **反思** 是所有维度的共同底层机制
- **资源** 为所有维度提供能量支撑

### 关键连接

| 连接 | 说明 |
|------|------|
| 方向 → 行动 | 目标分解为任务，任务填充日程 |
| 行动 → 资源 | 执行需要能量，健康状态影响执行能力 |
| 资源 → 认知 | 清晰的头脑提升学习效率 |
| 认知 → 行动 | 知识指导更有效的行动方法 |
| 反思 → 方向 | 复盘调整方向，确保方向仍然有效 |
| 反思 → 行动 | 复盘优化行动策略 |

## 应用原则

### 原则一：平衡优先于极端

五个维度需要保持动态平衡。过度的投入某一维度会导致其他维度失衡。

```
典型失衡：
- 疯狂工作，忽视健康 → 精力耗尽
- 追求知识，忽视行动 → 学而无用
- 设定目标，忽视反思 → 盲目奔跑
```

### 原则二：闭环运行

每个维度内部形成"计划→执行→复盘→改进"的闭环。

```
GTD循环：
收集 → 处理 → 组织 → 执行 → 回顾

习惯循环：
触发 → 行为 → 奖励 → 渴求
```

### 原则三：渐进演化

框架本身也需要随着认知升级而迭代。

- 每季度审视框架的有效性
- 根据实践调整子维度
- 保持框架的灵活性

## 与 MeOS 的关系

五维度框架是 MeOS 的理论基石，MeOS 通过数字化工具将框架落地为具体功能。

| 维度 | MeOS 功能模块 |
|------|---------------|
| 方向 | 愿景板、领域管理、目标设定、心态练习 |
| 行动 | 待办列表、习惯追踪器、日历管理 |
| 认知 | 课题管理、洞察库、阅读列表 |
| 反思 | 每日复盘模板、周期复盘工具 |
| 资源 | 订阅管理、人脉图谱、健康仪表盘 |
