## 二、整合策略

### 2.1 今日合并仪表盘 ✅

**整合逻辑**：
- 仪表盘的核心价值是"一眼看全局"，但它本身不产生行动
- 今日页面的核心价值是"告诉我该做什么"
- 将仪表盘的统计概览（活跃目标数、待办数、习惯进度、反思状态、愿景横幅）嵌入今日页面顶部

**改动点**：
- `Today.tsx`: 顶部增加"今日概览"栏，包含活跃目标、待办、活跃课题、反思状态四张统计卡片，以及愿景横幅
- `App.tsx`: `/` 路由改为渲染 Today，`/today` 重定向到 `/`
- `Layout.tsx`: 左侧导航移除"仪表盘"独立入口，"今日"作为唯一首页入口

### 2.2 认知库合并四模块 ✅

**整合逻辑**：
- 笔记和洞察共用 `/insights` API，本质上是同一数据的不同展示方式
- 阅读是输入源管理，课题是研究容器，两者是"输入→处理"的关系
- 四个模块合在一起，形成统一的"第二大脑"工作空间

**页面内部结构**：
```
认知库 (/cognition/hub)
├── Tab: 笔记           ← 原 Notes.tsx
├── Tab: 阅读           ← 原 Reading.tsx
├── Tab: 课题           ← 原 Topics.tsx
└── Tab: 洞察           ← 原 Insights.tsx
```

**改动点**：
- 新建 `pages/CognitionHub.tsx`: 顶部 Tab 切换四个认知模块
- `Layout.tsx`: 认知维度改为"认知库"一个入口
- `App.tsx`: 四个旧路由重定向到 `/cognition/hub?tab=xxx`
