# 行动维度：待办 (Tasks)

> "完美的工具是不存在的，真正重要的是使用工具的人。" —— 安东尼·马斯

## 待办管理的本质

待办是行动的基本单元，是将意图转化为成果的桥梁。一个混乱的待办系统会产生焦虑、遗漏和低效；一个清晰的待办系统能带来掌控感、专注力和持续的进展。

**待办管理的核心挑战**：
- 收集：如何确保不遗漏任何需要做的事？
- 处理：如何快速判断每个待办的下一步行动？
- 组织：如何让待办易于查找和执行？
- 执行：如何在众多待办中选择做什么？

### 第一阶段：收集 (Collect)

**收集原则**：
- 将所有"需要做"的事情从大脑转移到外部系统
- 保持收集工具的单一性，避免散落在多个地方
- 清空收集工具，而非让它成为存储场所

**收集触发点**：
```
日常触发：
- 邮件中的待办事项
- 会议中决定要做的事
- 别人请求你做的事
- 自己的想法和承诺
- 看到的需要处理的信息

定期触发：
- 每日晨间收集
- 每日结束时收集
- 每周回顾时全面收集
```

**收件箱 (Inbox) 设计**：
```
收件箱 = 所有"需要处理"但尚未分类的事项

要求：
1. 单一入口：所有事项都进入收件箱
2. 定期清空：收件箱不是存储箱
3. 快速捕获：捕获动作要简便

工具示例：
- 纸质笔记本的"收集页"
- 任务App的"收件箱"视图
- 邮件的"待处理"文件夹
```

---

### 第二阶段：处理 (Process)

**处理的核心问题**：对于每个收集箱中的事项，做出决定。

**2分钟原则**：
```
如果某事能在2分钟内完成，立即执行，不要放入系统。

示例：
- 回复简单邮件（<2分钟）→ 立即回复
- 整理文件（<2分钟）→ 立即归档
- 简单确认（<2分钟）→ 立即联系
```

**决策树 (Decision Tree)**：

```
收集箱中的一个待办事项

    ↓
这是可以执行的动作吗？
    ↓
否 → 垃圾桶
    - 不需要做
    - 信息仅供参考

是 ↓

2分钟内能完成吗？
    ↓
是 → 立即执行
    ↓
否 ↓

这件事需要多步才能完成吗？
    ↓
是 → 归入"项目"清单
    ↓ 为下一步行动
否 ↓

下一步行动是什么？
    ↓
有明确的下一步 → 归入情境清单
无明确的下一步 → 等待他人完成/等待条件
```

**处理注意事项**：
```
1. 一件事只做一次决策
   - 不要反复"再考虑"
   - 决策后立即执行或归档

2. 保持客观
   - 不要因为喜欢/讨厌而推迟决策
   - 基于"是否需要做"而非"想做"

3. 向前推进
   - 目标是"完成"不是"收集"
   - 每个事项都需要最终出口
```

### 第三阶段：组织 (Organize)

**组织结构**

```
┌─────────────────────────────────────────────────────┐
│              待办组织系统                          │
├─────────────────────────────────────────────────────┤
│                                                     │
│  项目清单                                           │
│   └─ 项目A                                          │
│      ├─ 待办1（下一步行动）                        │
│      ├─ 待办2                                       │
│      └─ 待办3                                       │
│   └─ 项目B                                          │
│      └─ ...                                         │
│                                                     │
│  情境清单                                           │
│   ├─ @工作 → 工作中可执行的                        │
│   ├─ @电话 → 需要打电话的                         │
│   ├─ @电脑 → 需要电脑的                           │
│   ├─ @外出 → 外出时处理的                         │
│   └─ @等待 → 等待他人的                          │
│                                                     │
│  日历                                               │
│   └─ 有具体日期的行动                               │
│                                                     │
│  也许/以后                                          │
│   └─ 将来想做但不是现在的                         │
│                                                     │
└─────────────────────────────────────────────────────┘
```

**项目定义**
```
项目 = 任何需要多个步骤才能完成的事

判断标准：
- 能在一个季度内完成吗？
- 需要不止一个行动吗？
- 有明确的完成状态吗？

示例：
项目：完成产品发布 ✓
项目：改善沟通能力 ✓（需要设定具体边界）
非项目：做更多运动（模糊，需要具体化）
```

**情境标签设计**
```
常见情境分类：

地点导向：
- @办公室
- @在家
- @外出
- @购物

工具导向：
- @电脑
- @电话
- @邮件
- @会议

精力导向：
- @高精力：复杂任务
- @低精力：简单机械任务

时间导向：
- @5分钟：微任务
- @25分钟：番茄钟任务
```

**优先级设置**

```
优先级分类（建议简化为3级）：

P0 - 必须做（今天必须完成）
P1 - 应该做（今天尽量完成）
P2 - 可以做（有时间再做）

注意：
- 不要过度细分优先級（3级足够）
- 优先级需要每日重新评估
- P0不超过3个
```

### 第四阶段：回顾 (Review)

**回顾节奏**

```
每日回顾（5-10分钟）：
- 清空收集箱
- 确认今日P0任务
- 调整时间安排
- 处理突发情况

每周回顾（30-60分钟）：
- 清空所有收集箱
- 回顾所有活跃项目
- 回顾日历与下周安排
- 回顾习惯追踪器
- 清理与整理系统
```

**每周回顾清单**

```
┌─────────────────────────────────────────────────────┐
│              每周回顾清单                           │
├─────────────────────────────────────────────────────┤
│ □ 清空所有收集箱（邮件、笔记、实物）              │
│ □ 回顾所有项目状态（完成/进展/停滞）              │
│ □ 确认下周日历上的承诺                            │
│ □ 设定下周TOP3目标                                │
│ □ 检查习惯追踪器状态                              │
│ □ 处理积压的"以后再做"清单                       │
│ □ 清理数字空间（桌面、下载、收件箱）             │
│ □ 庆祝上周的成就                                  │
└─────────────────────────────────────────────────────┘
```

### 第五阶段：执行 (Execute)

**执行时的选择框架**

```
情境 × 精力 × 时间 → 匹配最佳任务

步骤：
1. 评估当前时间有多少
2. 评估当前精力状态
3. 评估当前情境（地点、工具）
4. 选择匹配的任务

"吃青蛙"策略：
先完成最困难/最重要的任务
早晨精力最佳时做P0任务
```

**情境匹配示例**

```
情境：早晨9点，办公室，高精力，有1小时
→ 选择：P0高难度任务，深度工作

情境：下午2点，在家，低精力，有30分钟
→ 选择：P2机械任务，整理/归档

情境：等待会议开始，有10分钟
→ 选择：快速邮件回复，阅读资料

情境：通勤路上，有20分钟
→ 选择：@外出情境的任务，学习音频
```

## 待办系统常见问题

### 问题1：待办列表太长

**原因**：收集过度，处理不足

**解决方案**：
```
1. 批量处理：留出整块时间处理待办
2. 坚定删除：不需要做的立即删除
3. 委托出去：不是必须自己做的就转交
4. 推迟明确：不能立即做的，设定具体执行日期
```

### 问题2：遗漏重要事项

**原因**：收集机制有漏洞

**解决方案**：
```
1. 审查收集入口：哪些地方还没覆盖？
2. 每日回顾：确认没有遗漏
3. 周全检查：每周检查所有潜在来源
4. 养成习惯：快速捕获任何"需要做"的想法
```

### 问题3：无法决定下一步

**原因**：待办太模糊

**解决方案**：
```
1. 追问"下一步行动是什么？"
2. 如果不知道，就先研究/探索
3. 分解成更小的步骤
4. 给每个待办设定明确的触发条件
```

### 问题4：等待中的任务失控

**原因**：没有追踪"等待他人"的任务

**解决方案**：
```
1. 建立"等待"清单
   - 所有等待他人完成的任务归入此处
   - 标注：等待谁？预计什么时候？

2. 设置提醒
   - 超过3天没进展的要主动跟进
   - 临近预计时间要确认

3. 定期清理
   - 每周检查等待清单
   - 及时升级或放弃

等待任务追踪示例：
┌─────────────────────────────────────────────────────┐
│              等待清单                              │
├─────────────────────────────────────────────────────┤
│ 任务             │ 等待谁   │ 日期    │ 状态      │
├─────────────────────────────────────────────────────┤
│ 设计稿确认       │ UI设计师 │ 5/15    │ ⚠ 超期   │
│ 客户反馈         │ 张总     │ 5/18    │ 进行中    │
│ 合同签署         │ 法务     │ 5/20    │ 正常     │
│ 预算审批         │ 财务     │ 5/12    │ ✓ 已完成 │
└─────────────────────────────────────────────────────┘

跟进原则：
- 等待不是放弃，是主动管理
- 超过预期时间要温和跟进
- 必要时升级或重新分配
```

### 问题5：委托的任务丢失

**原因**：委托出去后没有追踪

**解决方案**：
```
1. 委托时明确：
   - 任务内容是什么？
   - 预期什么时候完成？
   - 如何汇报结果？

2. 记录委托：
   - 在任务备注中标注"委托给XXX"
   - 设置提醒检查进展

3. 主动检查：
   - 不定期询问进展
   - 在预期时间确认结果

委托任务模板：
┌─────────────────────────────────────────────────────┐
│ 委托任务记录                                        │
├─────────────────────────────────────────────────────┤
│ 任务：完成Q2报告数据分析                           │
│ 委托给：小李                                       │
│ 委托时间：5月8日                                   │
│ 预期完成：5月15日                                  │
│ 汇报方式：完成后发消息                             │
├─────────────────────────────────────────────────────┤
│ 进展记录：                                         │
│ 5/10 - 小李说数据获取有困难，正在解决             │
│ 5/13 - 小李说周三前能完成                         │
│ 5/15 - ✓ 完成                                     │
└─────────────────────────────────────────────────────┘
```

## 最佳实践

### 实践1：每日开始仪式

```
早晨执行：
1. 打开待办系统（不是邮件）
2. 确认今日P0任务（≤3个）
3. 安排时间块
4. 设定意图："今天我要完成..."
```

### 实践2：边界保护

```
不要让待办无限膨胀：

规则：
- 每日新增待办不超过5个
- 每周清理一次"死待办"
- 无法决定时就问："我真的需要做吗？"
```

### 实践3：完成庆祝

```
完成重要任务后：
- 在待办系统标记完成
- 记录完成时间（自我激励）
- 思考下一步
- 小休息，奖励自己
```
