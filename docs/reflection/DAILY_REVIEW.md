# 反思维度：每日反思 (Daily Review)

> "复盘是向前看，而不是向后看。" —— 温斯顿·丘吉尔

---

## 每日反思的本质

每日反思是持续迭代的基础单元。它不是流水账式的记录，而是对一天经历的提炼与升华。

**每日反思的核心价值**：

```
早晨 → 设定意图：我今天要完成什么？
白天 → 执行与记录：发生了什么？
夜间 → 反思与调整：我学到了什么？明天要做什么？
```

## 每日反思的结构

### 模块一：今日概览

```
┌─────────────────────────────────────────────────────┐
│ 【今日概览】                                       │
├─────────────────────────────────────────────────────┤
│ 日期：2026-05-11 星期日                            │
│ 天气：晴朗                                        │
│ 整体评分：7/10                                     │
│                                               　　 │
│ 今日关键词：专注、挑战、成长                      │
│                                               　　 │
└─────────────────────────────────────────────────────┘
```

**评分维度参考**：

```
评分因素：
- 目标达成度
- 能量状态
- 人际关系
- 情绪稳定性
- 成长感

评分标准：
10分：完美的一天，几乎没有可以改进的地方
8-9分：很好的一天，有小瑕疵但整体满意
6-7分：普通的一天，没有特别的好或坏
4-5分：挑战的一天，有较多困难但有所学
3分以下：艰难的一天，需要记录并分析原因
```

```

├─────────────────────────────────────────────────────┤
│                                               　　 │
│ 1. 完成项目A的核心模块开发                        │
│    └─ 这是我本周最重要的目标达成                   │
│                                               　　 │
│ 2. 完成了一次高质量的深度工作                    │
│    └─ 连续3小时专注，产出了可用的初稿            │
│                                               　　 │
│ 3. 与老朋友深度长谈，重建连接                     │
│    └─ 关系维护很重要，这个投入很值得              │
│                                               　　 │
└─────────────────────────────────────────────────────┘
```

**成就记录原则**：

```
1. 具体而非笼统
   - ✗ "工作做得不错"
   - ✓ "完成Q2报告初稿，获得老板认可"

2. 包含为什么重要
   - 每个成就后面标注为什么值得记录

3. 庆祝进步
   - 即使是小事也值得记录
   - 累积成就是自信的基础
```

### 模块三：教训记录

```
┌─────────────────────────────────────────────────────┐
│ 【今日教训】                                       │
├─────────────────────────────────────────────────────┤
│                                               　　 │
│ 1. 下午的时间被紧急会议打断                       │
│    └─ 问题：没有保护核心工作时间的机制           │
│    └─ 教训：需要在日历上明确标记深度工作时间      │
│                                               　　 │
│ 2. 运动计划因时间不足取消                         │
│    └─ 问题：低估了其他事情的时间需求              │
│    └─ 教训：运动需要作为硬性日程，而非弹性选项   │
│                                               　　 │
│ 3. 回复邮件时过于仓促                             │
│    └─ 问题：没有仔细思考就回复了                  │
│    └─ 教训：重要邮件需要"冷却期"，不要立刻回复   │
│                                               　　 │
└─────────────────────────────────────────────────────┘
```

**教训记录原则**：

```
1. 不责备自己
   - 教训不是自责，是学习
   - 关注"我能学到什么"而非"我犯了什么错"

2. 寻找模式
   - 这个教训是否之前也出现过？
   - 这是系统性问题还是偶然事件？

3. 转化为行动
   - 每个教训都要有对应的改进行动
   - 不然教训只是抱怨
```

## 每日反思执行指南

### 时间选择

```
推荐时间：夜间睡前15-20分钟

优点：
- 一天完整结束，有完整的视角
- 帮助大脑整理一天的信息
- 为次日设定清晰的意图
- 睡前复盘有助于潜意识处理

替代方案：
- 傍晚（18:00-19:00）
  优点：记忆鲜活
  缺点：可能被打断，视角不够完整

不推荐：
- 早晨（前一天的总结不完整）
- 工作中间（打断工作流程）
```

### 执行原则

```
1. 固定时间，形成仪式
   - 每天同一时间执行
   - 形成习惯后不会消耗意志力

2. 写下来，而非只是想
   - 写的过程是整理思考的过程
   - 文字记录便于后续回顾

3. 简短但完整
   - 控制在15-20分钟
   - 关注质量而非数量

4. 重点在于洞察与改进行动
   - 不是流水账，是提炼
   - 每个模块都要产出可执行的结论

5. 接受不完美
   - 有时确实没有那么多值得记录的
   - 简短也可以是好反思
```
